## Night: Bug Fixes & Testing

After implementing the UI changes, ran into several issues during web export testing:

### Issues Fixed
1. **Emoji encoding** - Web export couldn't render Unicode emojis (⚔️, 🔒, •, etc.) - replaced with plain text
2. **AvatarData property errors** - `ability_name` doesn't exist directly on AvatarData, it's in `abilities[0].ability_name`. Also `is_nft` should be `is_nft_avatar`
3. **CardManager missing functions**:
   - `_draw_cards(n)` → loop calling `_draw_card()` n times
   - `_can_attack(card)` → manual check for `META_HAS_ATTACKED` and `META_SUMMONED_THIS_TURN`
   - `_execute_avatar_attack()` → `_execute_face_attack(attacker, false)`
   - `_execute_character_attack()` → `_execute_attack(attacker, defender)`

### Testing Notes
- F5/F12 don't work in browser when Godot canvas has focus (captures input)
- Browser caches aggressively - use Incognito or clear cache when testing exports
- Python HTTP server shows 304 (cached) vs 200 (fresh) - watch for this

**Final result: Game loads and works!** 🎮✅
